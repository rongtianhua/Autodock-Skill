# API 字段索引对照表

## 重要教训

**不同 API 的字段索引必须分别验证，绝不能假设一致！**

## 腾讯 API vs 新浪 API 五档盘口索引对比

**股票示例**：贵州茅台（600519）

| 档位 | 腾讯索引 | 新浪索引 | 腾讯值 | 新浪值 |
|------|---------|---------|--------|--------|
| bid1 价格 | 9 | **11** | 1446.88 | 1446.88 ✅ |
| bid1 数量 | 10 | 10 | 1 | 100 |
| bid2 价格 | 11 | **13** | 1446.81 | 1446.81 |
| bid3 价格 | 13 | **15** | 1446.80 | 1446.80 |
| bid4 价格 | 15 | **17** | 1446.79 | 1446.79 |
| bid5 价格 | 17 | **19** | 1446.78 | 1446.78 ✅ |

**验证结果**：两者 bid5 完全一致（1446.78），证明新浪索引修正是正确的。

## 常见错误示例

### 错误代码（将当前价误当 bid5）
```python
# 错误：parts[7] 是当前价 1446.90，不是 bid5
sina_bid5_index = [11, 13, 15, 17, 7]  # ❌ parts[7]=1446.90
```

### 正确代码
```python
# 正确：五档盘口使用连续索引 [11,13,15,17,19]
sina_bid5_index = [11, 13, 15, 17, 19]  # ✅ parts[19]=1446.78
```

## 调试方法

1. 先打印原始数据：`print(parts)` 查看每个位置的原始值
2. 对比两个 API 的相同字段（bid1 价）确认索引对应关系
3. 验证最高档（bid5）数值是否符合逻辑（应低于或等于当前价）

## 腾讯 API 字段索引参考

```python
# 腾讯五档盘口索引
price_idx = [9, 11, 13, 15, 17]  # bid1-5 价格
vol_idx = [10, 12, 14, 16, 18]    # bid1-5 数量
```

## 新浪 API 字段索引参考

```python
# 新浪五档盘口索引
bid_price_idx = [11, 13, 15, 17, 19]  # bid1-5 价格（注意不是 [11,13,15,17,7]）
bid_vol_idx = [10, 12, 14, 16, 18]     # bid1-5 数量
ask_price_idx = [21, 23, 25, 27, 29]  # ask1-5 价格
ask_vol_idx = [20, 22, 24, 26, 28]    # ask1-5 数量
```
