## Feishu Bridge 迁移与配置

### 完成的工作

1. **安装 feishu-bridge 技能**
   - 从 ClawHub 安装了 feishu-bridge (v1.0.0)
   - 位置: `/Users/allenrong/.openclaw/workspace/skills/feishu-bridge/`

2. **停用内置 feishu 插件**
   - 修改 `~/.openclaw/openclaw.json`: `channels.feishu.enabled = false`
   - 重启 Gateway 生效

3. **配置 feishu-bridge**
   - 创建 `~/.clawdbot/clawdbot.json` 网关配置
   - 创建 `~/.clawdbot/secrets/feishu_app_secret` 存储密钥
   - 修改 `bridge.mjs` 添加 `operator.write` 权限:
     ```javascript
     scopes: ['operator.read', 'operator.write']
     ```

4. **配置开机自启动**
   - 创建 LaunchAgent: `~/Library/LaunchAgents/com.clawdbot.feishu-bridge.plist`
   - 服务 PID: 51413
   - 开机自动启动，崩溃自动重启

### 架构变化

```
之前: 飞书云 → OpenClaw 内置插件 (WebSocket) → AI
现在: 飞书云 ←→ feishu-bridge (本地) ←→ OpenClaw Gateway ←→ AI
```

### 管理命令

```bash
# 查看状态
launchctl list | grep feishu

# 查看日志
tail -f ~/.clawdbot/logs/feishu-bridge.err.log

# 手动停止
launchctl unload ~/Library/LaunchAgents/com.clawdbot.feishu-bridge.plist

# 手动启动
launchctl load ~/Library/LaunchAgents/com.clawdbot.feishu-bridge.plist
```

### 注意事项

- feishu-bridge 使用 WebSocket 长连接，无需公网 IP
- 群聊中仅当 @mention、消息以 ? 结尾、或包含请求动词时回复
- operator.write 权限允许主动推送消息（如定时任务）
