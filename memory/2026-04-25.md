# 2026-04-25 日志

## 分子对接环境重建（autodock → autodock313）

### 背景
autodock (py311) 环境里没有 PyMOL Python 模块（pymol-open-source 无 py311 构建），导致分子对接分析的 3D 全景图和口袋放大图空白。

### 解决：新建 autodock313 环境（Python 3.13）

**创建过程：**
```bash
conda create -n autodock313 python=3.13 -y
# pip 先装所有通用依赖
# conda 装 pymol-open-source（自动选 py313 构建）
# conda 装 rdkit vina numpy scipy matplotlib biopython pandas 等
# pip 装 meeko propka pdb2pqr biotite prody nglview 等
# pip 装 gemmi（cp313 wheel 有）
```

**环境信息：**
- 路径：`/opt/homebrew/Caskroom/miniconda/base/envs/autodock313/`
- 大小：2.1GB
- Python：3.13.13

**激活方式：**
```bash
conda activate autodock313
```

### 两个环境对比

| 特性 | autodock (py311) | autodock313 (py313) |
|------|-----------------|---------------------|
| pymol | ❌ 无 | ✅ 3.1.0 |
| RDKit | 2023.9.6 | **2026.03.1** |
| Vina | 1.2.5 | 1.2.5 |
| openbabel | ✅ | ❌ 无 py313 |
| plip | ✅ | ❌ 无 py313 |
| 其他生信包 | 全部有 | 全部有 |

### autodock313 能导入的包（23个）
pymol ✅ / rdkit ✅ / vina ✅ / matplotlib ✅ / numpy ✅ / scipy ✅ / biopython ✅ / pandas ✅ / meeko ✅ / propka ✅ / pdb2pqr ✅ / biotite ✅ / prody ✅ / nglview ✅ / gemmi ✅ / plotly ✅ / seaborn ✅ / PyRx ✅ / dockstring ✅ / pillow ✅ / reportlab ✅ / biotraj ✅ / lxml ✅

### 无法安装的包
- **openbabel**：pip 无 py313 wheel，源码编译依赖系统 Open Babel C++ 库，Homebrew 无此 formula
- **plip**：依赖 openbabel，故也跟着装不了

### 适用范围

**autodock313（默认，下次跑分子对接分析用这个）：**
- 蛋白质-小分子对接核心流程
- PyMOL 3D 渲染（`from pymol import cmd; cmd.png()` 输出 292KB 正常图）
- 配体准备（meeko/RDKit 替代 openbabel）
- 对接打分、结果分析

**autodock (py311)（保留，特殊场景用）：**
- PLIP 蛋白-配体相互作用分析
- openbabel 格式转换（但 RDKit 可替代大部分功能）

### 教训
- pymol-open-source conda 构建只支持 py38/py39/py313，不支持 py311/py312
- 遇到"之前正常突然断了" → 先查 PyPI/conda 新版本（baostock 的教训 + pymol 的教训）
