## 第一轮：审查与修复（6项问题）

### 修复1：clash_threshold 默认值调整
- **问题**：clash_threshold 默认 0.5Å，但 explicit-H 系统下 H-H 接触天然产生更大 overlap
- **修复**：默认 0.5Å → **1.2Å**（基于 VDW 半径和分子动力学文献）
- **新增**：`tests/test_clash.py`（8个测试，全面覆盖）

### 修复2：误导性注释修正
- **问题**：`_core.py` 中 `object.__setattr__` 注释与 slots dataclass 无关
- **修复**：改为 `# Set output paths (fields already defined in DockingResult dataclass)`

### 修复3：P2Rank + Java 可用性检测
- **问题**：`cmd_status()` 未检测 P2Rank 和 Java
- **修复**：增加检测逻辑，status 输出显示 Java 和 P2Rank 状态

### 修复4：日志增强
- **新增**：`--log-file` CLI 参数
- **新增**：自动检测 `~/.openclaw/logs/` 目录启用 FileHandler

### 修复5：DrugBank 调研与替代方案
- **结论**：DrugBank 公开 API 已关闭，学术下载暂停
- **新增**：`DATA_SOURCES_AUDIT.md` + `DRUGBANK_ALTERNATIVES.md`
- **决策**：暂不实现 DrugBank 解析模块，等用户下载 XML 后再说

### 修复6：死代码清理 + ZINC22 解析完善
- **清理**：`_init_vina_worker()` 嵌套函数（已不被调用）
- **修复**：ZINC22 `parse_tranche_props()` 缺失属性返回 `None` 而非 `0`
- **修复**：超时线程 `daemon=True` → `daemon=False`（避免 C++ 内存损坏）

## 第五轮：SwissModel 增强同源建模

### 功能1：fetch_protein_swissmodel() 增强
- 质量评分提取：GMQE、QMEAN、identity、coverage
- 智能排序：`score = GMQE × coverage × (1 + identity)`
- `provider_filter`: `'any'` | `'swissmodel'` | `'pdb'`
- `min_coverage` / `min_identity` 阈值过滤
- `return_all`: 多模型下载 + 完整质量报告

### 功能2：fetch_protein_swissmodel_advanced()（NEW）
- 全自动质量筛选 + 质量评级（excellent/good/moderate/poor）
- 阈值配置：`min_gmqe`、`min_qmean`、`require_full_coverage`
- AlphaFold DB 自动回退
- 返回 `all_candidates` 列表供用户审查

### 功能3：SwissModel REST API Token 管理
- `swissmodel_get_token()`: 获取/缓存 Token（`~/.openclaw/swissmodel_token.json`）
- `swissmodel_clear_token()`: 清除缓存
- `swissmodel_submit_alignment()`: 提交建模任务（需预先知道模板）
- `swissmodel_check_status()`: 查询任务状态
- `swissmodel_download_result()`: 下载结果
