### P0 修复（严重 Bug）

#### 1. `detect_interactions_plip` 被定义两次（`_interactions.py`）
**位置：** 第461行（第一个定义）和第846行（第二个定义，实际生效）
**问题：** Python 静默覆盖第一个定义，第一个定义内部包含约380行 dead code，其中有逻辑错误的内部函数分支。
**修复：** 删除第461–759行的第一个定义，只保留第2个。
**验证：** AST 解析后只剩1个定义，语法正确。

#### 2. `_build_complex_pdb_for_plip` GLY/TYR 硬编码（`_interactions.py`）
**位置：** 第351–357行
**问题：** 硬编码 `('GLY A 501' in l or 'TYR A 502' in l)` 仅针对 6LU7 这一个 PDB。换其他含共结晶配体的 PDB（如 HIV-1 蛋白酶 1BVE 含 JG1），PLIP 会再次把共结晶配体误识别为待分析配体。
**修复：** 通用化为标准氨基酸集合（20种 + MSE），将所有标准氨基酸的 HETATM 转为 ATOM。保留非标准 HETATM（辅因子/金属离子/共结晶抑制剂），使 PLIP 仍能检测 metal complexes。
```python
_STD_AA_HETATM = {'ALA','ARG','ASN','ASP','CYS','GLN','GLU','GLY',
                  'HIS','ILE','LEU','LYS','MET','MSE','PHE','PRO',
                  'SER','THR','TRP','TYR','VAL'}
```

### P1 修复（中等问题）

#### 3. `screen_ligands` → `composite_summary` 传参类型不匹配（`_docking.py`）
**问题：** 传入 PDBQT 文件路径列表给 `composite_summary`，但后者期望 PNG 图片路径（内部用 `mpimg.imread()`）。
**修复：**
1. 参数名 `titles` → `panel_titles`（匹配 `composite_summary` 签名）
2. 传入前先调用 `render_ligand_2d()` 生成 2D PNG，再传 PNG 路径

#### 4. `_render_svg_vector` 3D→2D 坐标错误（`_interactions.py`）
**问题：** `prot_c` 是 3D 蛋白质坐标 (x, y, z)，但直接取前两个分量传给 `GetDrawCoords`（期望 2D 分子坐标），导致线条从错误位置开始（z 维度被忽略，可能偏离数十埃）。
**修复：** 删除 `prot_c` 直接传入 `GetDrawCoords` 的错误逻辑。统一使用 dummy atom 的 2D 位置（RDKit 布局已自动放置在配体附近）作为所有相互作用线的锚点。

#### 5. `virtual_screen` seed 策略不一致（`_docking.py`）
**问题：** 每个化合物调用 `_get_vina_seed(seed)`，若 `seed=None`（随机模式），每个化合物得到不同随机种子，发表级可重复性不标准。
**修复：**
- 预计算 `base_seed = _get_vina_seed(seed)`
- `seed=None` 时：每个化合物使用 `base_seed + idx`（独立采样但可记录复现）
- `seed=int` 时：所有化合物共享同一固定种子

### 修复详情

#### 问题1：explicit-H 系统 clash 阈值修正
**科学背景：**
- 本技能使用 `removeHs=False`（explicit-H），氢原子参与 VDW 计算
- H 的 VDW 半径 1.20Å，两个 H 靠近时 overlap 很容易 > 0.5Å
- PoseBusters 的 0.5Å 标准基于 **heavy-atom-only**（不含 H）
- METTL8 + UrolithinA 实测 clash=1.24Å，按 0.5Å 阈值会被错误排斥

**修改（6处）：**
| 文件 | 位置 | 修改 |
|------|------|------|
| `_validation.py:237` | `clash_threshold=0.5` | → `1.2` |
| `_validation.py:244-257` | docstring | 增加 explicit-H (1.2Å) vs heavy-atom-only (0.5Å) 区分说明 |
| `_core.py:158` | 注释 `<= 0.5` | → `<= 1.2 (explicit-H threshold)` |
| `_docking.py:421` | docstring `clash_score < 0.5 Å` | → `clash_score < 1.2 Å for explicit-H` |
| `_docking.py:1442` | 示例 `0.5 keeps only clash-free` | → `1.2 for explicit-H systems` |

#### 问题2：误导性注释修正
- `_docking.py:1405`：`# Use object.__setattr__...` → `# Set output paths (fields already defined in DockingResult dataclass)`
- 原因：slots 问题实际通过在 dataclass 中添加字段解决，而非 `object.__setattr__`

#### 问题3：max_clash 示例更新
