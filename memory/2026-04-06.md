## OpenClaw 版本更新事故（重大事故 — 2026-04-06/07）

### 事故经过
1. 用户让我更新 OpenClaw
2. 我执行 `openclaw update`，成功下载新版本（2026.4.2 → 2026.4.5）
3. `openclaw doctor` 失败：大量依赖缺失（`@buape/carbon`, `@larksuiteoapi/node-sdk`, `@slack/web-api`, `grammy` 等）
4. 尝试逐个安装依赖，但需要 sudo 权限，无法通过 agent 完成
5. 最终导致 OpenClaw 完全宕机，命令行失效
6. **用户被迫重新运行官方一键安装脚本（完整重装），配置文件再次丢失**
7. 依靠 config-guardian 备份恢复了大部分配置，但花费了大量时间和精力（用户为此严重缺觉）

### 教训
1. **`openclaw update` 风险很高** — 大版本升级可能涉及依赖链大幅变化，不是简单的 `npm update`
2. **agent 无法修复系统级依赖问题** — 涉及 `sudo npm install` 时 agent 无能为力，只能让用户手动处理
3. **更新失败导致宕机的代价远超用户预期** — 从 "帮我更新下" 到 "重装+恢复配置+缺觉一整夜"
4. **config-guardian 救了一命** — 没有备份的话这次损失无法挽回
5. **openclaw 版本迭代快** — 2026.4.x 几乎每天都有小版本变化，大版本之间环境要求可能变化较大

### 行为准则更新
**今后用户要求更新 OpenClaw 时，必须先做风险评估：**
- 检查当前版本与目标版本的差距
- 检查是否有已知的依赖问题
- 如果跨越多个小版本或涉及大版本跳跃 → **强烈建议用户手动更新**
- 如果风险不确定 → **反馈风险并请求用户决定是否自己来**
- **绝不能在没有评估的情况下直接执行更新**
