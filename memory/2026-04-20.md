# 2026-04-20 日志

## R 环境调试总结

### 核心问题：conda R vs Homebrew R

**conda R (r-base 4.4.2) 局限性：**
- 编译器链与 macOS ARM64 不兼容（libgfortran/libgcc 路径结构损坏）
- 编译 R 包时 linker 找不到 `-lquadmath`，gcc 库路径指向不存在的目录
- 部分包（clusterProfiler/enrichplot/igraph）源码编译失败
- 适合场景：conda 环境里 Python↔R 桥接（reticulate）、轻量脚本

**Homebrew R (4.5.3) 优势：**
- Apple Clang + Homebrew gcc 工具链完整
- Java/rJava 配置正确（JNI 路径清晰）
- Bioconductor 包安装无障碍
- 是生物医学数据分析的主力 R

### 经验1：R 版本与包生态
- R 4.5.x（开发版）≠ 包生态不完整，只是 CRAN 发布滞后
- **CRAN 不支持时的解决方案：**
  - 从 GitHub 装开发版：`remotes::install_github("owner/repo")`
  - 今天成功：dmetar / TwoSampleMR / gsmr 均通过 GitHub 装上

### 经验2：poppler 缺失导致 bibliometrix 安装失败
- 错误：`fatal error: 'poppler-document.h' file not found`
- 解决：`brew install poppler` → pdftools 装好后 → contentanalysis → bibliometrix

### 经验3：conda R 的 libgfortran 修复尝试
- 修复命令：`mamba install --force-reinstall libgfortran5 gcc`
- 部分有效：meta/metafor 等基础包可正常加载
- 但 igraph 等需要编译的包仍然失败（链接器路径问题）
- **结论：conda R 编译链在 macOS ARM64 上结构性损坏，不值得继续修复**
