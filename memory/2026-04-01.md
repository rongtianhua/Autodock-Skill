## Thinking 默认级别 & 模型配置
- 用户再次遇到 thinking off → free router/qwen3.6-plus 需要 reasoning 的问题
- 根因：`agents.defaults.thinkingDefault` 没设，新会话/cron 任务默认 off
- 解决：`openclaw config set agents.defaults.thinkingDefault "medium"` → validate → gateway restart
- 经验教训：thinking 的 config key 是 `agents.defaults.thinkingDefault`（不是 thinking/think/thinkLevel），`openclaw config validate` 是唯一可靠的验证命令
- FreeRide 之前设好的模型配置完好无损

## 免费模型局限性讨论
- 用户反馈：思考过程外泄（只输出推理链无结论）、回答中断、多步工具调用流程中断，体感差
- **qwen3.6-plus-preview:free** 被认为是免费模型里体感相对较好的
- 免费模型根本问题：推理能力弱 + 速率限制 + 无 sandbox
- 改进建议分三层：
  1. **模型层（最有效）**：主力换成中小付费模型（如 kimi/kimi-code），免费模型仅作兜底
  2. **配置层**：固定 thinkingDefault、优化 fallback 链（别放 openrouter/free 在第一个，太随机）、调整 compaction
  3. **Agent 行为层**：回答前自检结论完整性、工具失败主动重试、长对话主动总结
