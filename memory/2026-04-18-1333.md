## 🔍 根本原因

MemOS 的 skill 创建是**两步分离**的：

```
第一步：Task Processor（任务检测）
每轮对话结束后自动运行
  → 分析对话内容
  → 发现有价值的任务 → 创建 task 记录（skill_status = NULL）
  → 同时在 skills 表创建一条记录（此时 quality_score = NULL）
  → 这些 skill 立即出现在 Dashboard 里

第二步：Skill Evolver（异步评估）
  → 捞取 skill_status = NULL 的任务
  → 评估是否值得生成正式 skill
  → 生成 → 调用 validator → 写入 quality_score
```

**67 个 skill 没有 quality_score = NULL 的原因**：第一步已执行（skill 已创建），但第二步 Skill Evolver 还没跑到它们。

## ⏳ 什么时候会有评分？

| skill_status = NULL | **89** | 待处理队列 |

| skill_status = 'not_generated' | 27 | ❌ 评估后判定不值得 |

**结论**：89 个任务在排队，Skill Evolver 会逐个处理。每完成一轮对话，它就会跑一个。**今天创建的 skill（比如 `memos-dreaming`）会在这批全部完成后才轮到评分。**
